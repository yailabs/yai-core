# Architecture Diagrams

This directory is the canonical home for architecture checkpoint diagrams.

These diagrams are target/checkpoint architecture aids. They are not proof
that a plane, backend, command, carrier or integration has been implemented.
Implementation truth remains in:

```text
docs/engineering/current-status.md
docs/engineering/four-repo-roadmap.md
docs/engineering/data-plane-roadmap.md
```

Official diagram assets should use stable names:

```text
yai-canonical-architecture.png
yai-data-plane-architecture.png
yai-operational-case-loop.png
yai-carrier-semantics.png
yai-formal-system-semantics.png
```

Source files, if kept, should use the same basename with the native extension.
Generated or exploratory image files must be renamed to the stable names before
they are committed.
