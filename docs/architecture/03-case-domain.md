# Case Domain

The case is the operational control domain.

A case binds subjects, policy, evidence, procedure posture and closure. It gives operations a bounded world in which control can be evaluated and effects can be reconstructed.

## Owns

```text
case identity
case boundary
case subject binding
case posture
case materialization
case evidence refs
case policy binding
case procedure record
case closure
```

## Does Not Own

```text
subject semantics
carrier execution
policy rule semantics
storage backend
graph reconstruction engine
memory consolidation engine
Console UX
API/SDK truth
```

## Case Loop

```text
open case
bind subjects
bind policy
receive material
evaluate attempts
link receipts
project state
close or continue
```

The case is not a workflow engine. Procedure records can exist, but workflow execution is not the core owner.
