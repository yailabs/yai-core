# Core Operational Spine

Status: SPINE.5 canonical operational spine.

YAI Core is not a list of modules.

YAI Core is a case-bound operational substrate that turns world participation
into residue, residue into data planes, data planes into live projections and
memory, and memory/projections into controlled future behavior.

The mature spine is:

```text
WORLD
-> RESIDUE
-> DATA
-> VIEW
-> CONTROL
-> MEMORY
-> RECONCILE
-> HOST
-> MODEL
-> OBSERVABILITY
-> EXTERNAL
```

It is not only:

```text
case -> subject -> op -> control -> receipt
```

## Macro-Spines

`WORLD`
: The case forms a bounded operational world.

`RESIDUE`
: Every meaningful participation in that world produces residue.

`DATA`
: Residue is stored, indexed, related, replayed, analyzed and materialized
through live/durable data planes.

`VIEW`
: The case becomes visible to models, agents, operators, APIs and audit through
controlled live projections.

`CONTROL`
: Policy, authority, obligations and gates decide which proposed operations can
become effects.

`MEMORY`
: Receipts, graph paths, decisions, policy and divergences become operational
memory.

`RECONCILE`
: Expected and observed state are compared; mismatches become explicit
divergence.

`HOST`
: Carriers are the only sanctioned bridge from case decision to host/world
effect.

`MODEL`
: Models are cognitive subjects that consume projections and emit
interpretations, claims or proposed attempts.

`OBSERVABILITY`
: The system measures whether the case remains fresh, causal, provenanced,
coherent, complete, replayable and useful.

`EXTERNAL`
: Interfaces, console and ai-environment orbit around yai-core without owning
core truth.

## Flow

```text
world formation
-> operational residue
-> live/durable data planes
-> controlled projection
-> model/operator/carrier behavior
-> receipt/evidence
-> graph/memory/reconcile
-> observability/evaluation
-> next projection/action
```

Every future implementation wave must declare which macro-spines and
subdeliveries it matures.
