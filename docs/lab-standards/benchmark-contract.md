# Benchmark Contract

Document type: `benchmark_report`.

Required sections:

```text
Benchmark Motivation
Systems Under Test
Workloads
Metrics
Experimental Setup
Results
Analysis
Threats to Validity
Reproducibility Package
```

## Section Requirements

Benchmark Motivation: explain why the benchmark exists.

Systems Under Test: identify systems, versions and relevant configuration.

Workloads: define inputs and task shapes.

Metrics: define units, measurement method and aggregation.

Experimental Setup: describe environment, commands and run controls.

Results: report measured values.

Analysis: interpret results without turning them into unsupported claims.

Threats to Validity: list internal, external, construct and conclusion risks.

Reproducibility Package: link artifacts needed to repeat or audit the run.

## Rule

Benchmark reports must keep raw output, normalized data, figures and final
interpretation distinct.
