# Document Structure Contract

The common section spine is:

```text
Problem
Claim or Question
Model / Definitions
Method
Result
Interpretation
Limitations
Artifacts
References
```

Not every document type uses every section. Every document type must declare
which sections are required.

## Section Roles

Problem: the technical issue, research gap or system question.

Claim or Question: the proposition being supported, or the question being
tested.

Model / Definitions: terms, abstractions, variables, systems or assumptions
needed to read the document.

Method: how observations or findings were produced.

Result: observations produced by the method.

Interpretation: meaning assigned to results or findings.

Limitations: boundaries where validity, transferability or confidence ends.

Artifacts: files, commands, data, configs, figures or references supporting the
document.

References: cited external or internal material.

## Rule

If a section is intentionally absent, the document type or vertical profile must
make that absence explicit.
