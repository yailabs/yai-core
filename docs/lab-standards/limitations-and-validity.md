# Limitations and Validity

Every scientific or technical document should state what its observations do
not prove.

## Internal Validity

Internal validity concerns whether the method supports the result inside the
observed setup.

Generic example:

```text
The command measured elapsed time, but background processes were not controlled.
```

## External Validity

External validity concerns whether the result transfers to other systems,
datasets, environments or users.

Generic example:

```text
The run used one host and may not transfer to other hardware or operating systems.
```

## Construct Validity

Construct validity concerns whether the metric measures the concept it claims
to measure.

Generic example:

```text
Token count was estimated from whitespace, so it is not equivalent to tokenizer output.
```

## Conclusion Validity

Conclusion validity concerns whether the interpretation follows from the data.

Generic example:

```text
Only one run was recorded, so variance and statistical confidence are unknown.
```
