# Visual Artifact Contract

A visual artifact is valid only when it has:

```text
id
type
title
source_run
source_data
generation_method
output_path
format
created_at
```

## Types

Allowed visual artifact types:

```text
chart
diagram
figure
table
topology
timeline
matrix
flow
```

## Formats

Allowed formats for now:

```text
svg
png
html
md
json
csv
```

## Preferences

Use SVG for static charts and diagrams.

Use HTML for interactive visuals.

Use Markdown for readable tables.

Use JSON or CSV for normalized data.

For generated lab figures, prefer reproducible Python generation with numpy,
pandas and matplotlib reading from `metrics.json` or declared run assets.

## Rules

- Every visual artifact must link to its source run.
- Every generated visual must declare source data.
- Every generated figure must appear in `report.md`; otherwise it should not be
  generated.
- Manual visuals must be marked with `generation_method: manual`.
- A report must link to the visual manifest, not only to image files.
- In compact runs, visual artifacts live under `runs/<run>/assets/` and must
  be listed in `manifest.json`.
- Test-level visual aggregates are regenerated by the lab-root
  `notebook.ipynb` and may be attached to a run only when the run report uses
  them.
- Do not create lab-level aggregate visual directories as the canonical source
  of truth.
- Prefer markdown tables to SVG figures for tiny categorical summaries.
- Presence/absence visuals are diagnostic evidence only; they are not benchmark
  figures.
