# Visual Review Checklist

Use this checklist before a report links generated visuals.

## Identity

- Artifact ID follows the run artifact convention.
- File path is inside the run folder.
- Manifest entry exists.
- Manifest ID and file path agree.

## Provenance

- Source run is declared.
- Source data is declared.
- Generation method is declared.
- Manual source is declared when applicable.

## Style

- Title is brief.
- Axes are labeled when axes exist.
- Units are declared when metrics are shown.
- Colors are intentional.
- Legend is present only when useful.
- No decorative 3D is used for quantitative charts.

## Interpretation

- Caption states what is shown.
- Caption does not overclaim.
- Limitations are stated when the visual could be misread.
- Report links back to data or manifest.
