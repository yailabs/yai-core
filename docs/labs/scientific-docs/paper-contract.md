# Paper Contract

This contract defines `position_paper` and `systems_paper`.

## Position Paper

Required sections:

```text
Problem
Position
Argument
Implications
Counterarguments
Limitations
References
```

Problem: state the issue.

Position: state the argument being defended.

Argument: explain support for the position.

Implications: describe consequences if the position is accepted.

Counterarguments: present serious objections.

Limitations: bound the position.

References: cite supporting material.

## Systems Paper

Required sections:

```text
Abstract
Introduction
Problem
Design Goals
System Model
Architecture
Implementation
Evaluation
Discussion
Limitations
Related Work
Conclusion
References
Appendix
```

Abstract: summarize the paper.

Introduction: introduce context and contribution.

Problem: state the system problem.

Design Goals: list goals and non-goals.

System Model: define actors, state, interfaces or assumptions.

Architecture: describe components and boundaries.

Implementation: identify implementation details and maturity.

Evaluation: report methods and results.

Discussion: explain implications.

Limitations: define validity boundaries.

Related Work: place the work relative to references.

Conclusion: summarize what is supported.

References: cite sources.

Appendix: include supplementary material.
