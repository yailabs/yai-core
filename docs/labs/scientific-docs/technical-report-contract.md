# Technical Report Contract

Document type: `technical_report`.

Required sections:

```text
Scope
System State
Method / Sources
Findings
Discussion
Limitations
Next Work
Artifacts
```

## Section Requirements

Scope: define what the report covers and excludes.

System State: record the observed implementation or repository state.

Method / Sources: identify source inspection, commands, reports, logs or other
inputs.

Findings: state technical observations.

Discussion: explain implications and tradeoffs.

Limitations: define incomplete areas and uncertainty.

Next Work: state follow-up work without pretending it is complete.

Artifacts: link source paths, command outputs, reports or snapshots.

## Rule

A technical report may contain findings without making research claims.
