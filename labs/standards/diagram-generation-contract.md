# Diagram Generation Contract

Diagrams are visual explanations of structure, flow, topology or relationships.

## Rules

- Use few nodes.
- Name relationships.
- Make direction clear.
- Avoid random icons.
- Do not place unsupported claims inside the diagram.
- Link the diagram to the source run or source document.
- Declare whether the diagram is generated or manual.

## Diagram Classes

```text
topology
flow
matrix
timeline
architecture sketch
state transition
```

## Provenance

Each diagram must be listed in a figure manifest or equivalent visual manifest.
The manifest must identify source data, source text or manual source.
